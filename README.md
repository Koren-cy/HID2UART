# HID2UART

Windows 平台下的 HID（键盘、鼠标、手柄）转 UART 串口桥接工具。

使用 PyQt5 构建桌面 GUI，捕获系统级 HID 事件并通过自定义二进制协议经串口发送，广泛用于嵌入式开发、遥控项目和人机交互研究。

## 功能特性

- **键盘事件捕获**：捕获所有按键的按下和释放事件，发送虚拟键码
- **鼠标事件捕获**：捕获鼠标移动、点击和滚轮事件，支持边缘夹取补偿
- **手柄事件捕获**：支持标准游戏手柄的按钮、摇杆和扳机
- **实时帧监视**：十六进制 + 可读描述双重视图
- **热插拔支持**：运行时自动刷新串口列表，支持手柄热插拔
- **可靠传输**：CRC-8 校验 + 帧头帧尾定界

## 系统要求

- Windows 操作系统
- Python 3.12+
- 物理或虚拟串口（用于连接外部设备）

## 安装

### 方式一：源码运行

```bash
# 克隆项目
git clone https://github.com/your-repo/hid2uart.git
cd hid2uart

# 创建虚拟环境（推荐）
python -m venv venv
venv\Scripts\activate

# 安装依赖
pip install -e .

# 运行
python main.py
```

### 方式二：仅安装运行时依赖

```bash
pip install -r requirements.txt
python main.py
```

### 开发环境安装

```bash
pip install -e ".[dev]"
```

## 快速开始

1. 连接串口设备到 PC
2. 运行 `python main.py`
3. 在左侧面板选择串口参数并点击「连接」
4. 勾选要捕获的输入类型（键盘/鼠标/手柄）
5. 点击「开始捕获」

## 通信协议

### 帧格式

```
┌────────┬──────┬─────┬────────┬──────┬────────┐
│ 帧头   │ 类型  │ 长度 │ 载荷  │ CRC  │ 帧尾   │
│ 0xFA55 │ 1字节 │ 1字节│ N字节 │ 1字节 │ 0x0D0A │
└────────┴──────┴─────┴────────┴──────┴────────┘
```

- **帧头**：固定 `0xFA55`，用于接收端帧同步
- **类型**：标识载荷内容类型
- **长度**：载荷字节数
- **载荷**：类型相关的数据
- **CRC**：类型 + 载荷的 CRC-8 校验和（多项式 0x07）
- **帧尾**：固定 `0x0D0A`

### 帧类型

| 类型值 | 名称 | 说明 |
|--------|------|------|
| 0x01 | KEYBOARD | 键盘按键事件 |
| 0x02 | MOUSE_BUTTON | 鼠标按键事件 |
| 0x04 | MOUSE_MOVE | 鼠标移动/滚轮事件 |
| 0x03 | GAMEPAD | 手柄按钮事件 |
| 0x05 | GAMEPAD_STICK | 手柄摇杆/扳机事件 |

### 帧示例

键盘按下 A 键（虚拟键码 0x41）：

```
FA 55 01 02 41 01 73 0D 0A
│  │  │  │  │  │  │  │  │
│  │  │  │  │  │  │  │  └─ 帧尾（1）
│  │  │  │  │  │  │  └──── 帧尾（0）
│  │  │  │  │  │  └──────── CRC-8
│  │  │  │  │  └─────────── 动作（按下=0x01）
│  │  │  │  └────────────── 虚拟键码 A=0x41
│  │  │  └───────────────── 载荷长度（2字节）
│  │  └───────────────────── 类型（键盘）
│  └───────────────────────── 帧头（1）
└───────────────────────────── 帧头（0）
```

## 开发

### 代码规范

- 遵循 PEP 8，使用 ruff 作为代码检查工具
- 所有公共 API 必须有类型提示
- 所有公共函数必须有中文 docstring
- 使用 `__all__` 显式声明模块公共 API

### 运行检查

```bash
# 代码格式化和检查
ruff check .

# 类型检查
mypy .

# 运行测试
pytest

# 带覆盖率测试
pytest --cov=. --cov-report=term-missing
```
